PCB-AOI: PCB 부품 배치 검출 및 자동화 품질 검사 시스템
본 프로젝트는 수동으로 진행되던 PCB 검사 공정을 자동화하여 휴먼 에러를 방지하고, 고속 공정 환경에서 신속하고 정확한 품질 판정을 내리기 위해 개발된 비전 기반 자동 광학 검사(AOI) 소프트웨어입니다. 외주 발주처인 (주)이원테크놀러지와의 협업을 통해 현장 실효성을 검증받았습니다.

1. 프로젝트 개요
수행 기간: 2024.01 ~ 2024.02

수행 방식: (주)이원테크놀러지 외주 연계 및 현대오토에버 모빌리티 SW 스쿨 실무 프로젝트

핵심 역할: 응용 소프트웨어 개발 및 UI/UX 리팩토링, 비전 검사 로직 모듈화 설계

2. 주요 특징 (Key Features)
🔍 3단계 교차 검증 알고리즘 (Hybrid Inspection)
단일 알고리즘의 한계를 극복하기 위해 세 가지 기술을 순차적으로 적용하여 검출 신뢰도를 99% 이상으로 확보했습니다.

HSV 색상 매칭: 부품의 고유 색상 영역을 분석하여 결측 및 오염 1차 필터링

이미지 차분(Diff) & Template Matching: 기준 이미지와의 기하학적 차이 분석으로 부품 뒤틀림 및 오차 검출

YOLO 기반 객체 검출: 딥러닝 모델을 통한 최종 다중 검증으로 검사 정확도 극대화

🛠 객체지향 기반의 유연한 아키텍처
로직 모듈화: InspectionTarget 클래스를 설계하고 상속과 다형성을 활용하여 신규 PCB 모델이나 부품이 추가되어도 코드 수정 없이 대응 가능한 구조를 구축했습니다.

동적 ROI 관리: 사용자 인터페이스를 통해 검사 영역(ROI)을 실시간으로 설정하고 관리할 수 있습니다.

👤 현장 수용성 최적화 UI/UX (Refactored)
작업자 중심 인터페이스: 기존의 복잡한 개발용 로그 출력을 지양하고, Pass/Fail 대형 인디케이터와 시각적 판정 결과를 대시보드화하여 작업자의 인지 부하를 최소화했습니다.

3. 기술 스택 (Tech Stack)
Language: Python 3.9+

GUI Framework: PyQt5

Computer Vision: OpenCV, YOLO (v8)

Data Management: SQLite, JSON (Configuration & Result Serialization)

Design Principle: OOP (Object-Oriented Programming)

4. 프로젝트 구조 (Project Structure)
Plaintext
├── algorithms/           # 비전 검사 알고리즘 (HSV, SIFT, Template Matching 등)
├── manager/              # 카메라 제어, DB 관리, 타겟 객체 관리 모듈
├── models/               # 알고리즘 및 검사 대상(InspectionTarget) 데이터 모델
├── ui/                   # PyQt5 기반 메인 윈도우 및 대시보드 레이아웃
├── utils/                # JSON 처리 및 좌표 변환 유틸리티
└── main.py               # 어플리케이션 실행 엔트리 포인트
5. 문제 해결 및 성과 (Problem Solving & Results)
UI 리팩토링을 통한 택타임(T/T) 사수
문제: 초기 버전의 로그 노출형 UI가 현장 작업자의 작업 속도를 저하시키고 실시간 대응을 방해함.

해결: 시각적 직관성을 높인 인디케이터 중심 UI로 전면 개편.

결과: 공정 표준 택타임 준수율 100% 달성 및 작업 편의성 극대화.

코드 모듈화를 통한 개발 리드타임 단축
문제: 검사 로직의 하드코딩으로 인해 신규 모델 도입 시 셋업 시간이 과다 발생함.

해결: 알고리즘과 인터페이스를 분리하는 객체지향 설계를 도입하여 플러그인 방식의 로직 추가 구현.

결과: 신규 공정 대응 리드타임 단축 및 피드백 수용 속도 25% 향상.

6. 기대 효과 및 회고
품질 안정화: 자동화 시스템 도입을 통해 공정 불량률 15% 감소에 기여.

엔지니어링 통찰: "현장에서 사용되지 않는 기술은 가치가 없다"는 원칙 아래, 사용자(작업자) 중심의 소프트웨어 설계가 스마트 팩토리의 핵심임을 체감함.
